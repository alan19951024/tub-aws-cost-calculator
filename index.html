🏦 背景說明

台灣OO銀行擁有超過 150 萬名個人與企業客戶，長期以傳統地端（On-Premises）資料中心運行核心業務系統。隨著數位金融浪潮衝擊，銀行決策層決定啟動「數位轉型計畫」，目標是將銀行的網路應用系統遷移至 AWS 雲端，以提升系統彈性、降低維運成本，並達到金管會要求的業務連續性（BCP）規範。

• Storage amount: 300 GB

STEP 5
新增 Amazon CloudFront
搜尋 CloudFront → Add to estimate
• Data transfer out to internet: 800 GB(10萬人次網頁造訪 X 8MB/次)
• Region：選 亞太地區
• HTTPS requests 可保留預設
STEP 6
新增 Elastic Load Balancing（ALB）
搜尋 Elastic Load Balancing → Add to estimate
• Load balancer type: Application Load Balancer
• Number of Application Load Balancers: 1
• Processed bytes（已處理的位元組）: 16 GB(500,000 × 32 KB) / month ，單位選「GB 每個月」
• Number of rules: 10（預設值即可）
• New connections per second: 1
• Avg processing time: 1 ms（預設）
• ⚠️ 「每個 ALB 每秒的平均請求數量」填 1，不要填 500,000（那是每月總量）
STEP 7

搜尋 VPC → Add to estimate
• Number of NAT Gateways: 1
• Data Processed per NAT Gateway: 80 GB / 月
• 【區域 NAT 閘道】為必填欄位，填入：
  – Number of Regional NAT Gateways: 1
  – Number of Availability Zone: 1
  – Data processed per Regional NAT Gateway: 0 GB（流量填0，僅產生固定費）
• ⚠️ 「區域 NAT 閘道」為必填，流量填 0 GB 即可，費用不會增加

各服務計費公式
🌐
Amazon Route 53
DNS · Global 服務
月費 = Hosted Zone 費 + Query 費
= $0.50/zone/月 + $0.40/百萬次
# 1 zone + 50萬 q = $0.50 + $0.20 = $0.70 ≈ $1
💡
計費重點
Route 53 費用極低，每月不到 $1。
主要功能是把 bank.com.tw 解析成 CloudFront 的 IP 位址（DNS 解析），不處理 HTTP 流量。
📐
架構說明
Route 53 與 CloudFront 並排在 Global 層（VPC 外）。
用虛線連接 CloudFront 代表「DNS 解析關係」，不是流量路徑。
☁️
Amazon CloudFront
網路 & CDN · $0.120/GB (Japan)
月費 = 流量(GB) × $0.120/GB
# 東京(Japan) $0.120/GB（AWS Pricing Calculator Japan 費率）
本題：800GB × $0.120 = $96
💡
計費重點
僅計算「傳出流量」(Data Transfer Out)。
東京(Japan) Region $0.120/GB，與台北費率相同，高於美國 $0.085。
🎯
計算機設定
Service: CloudFront → Data transfer out → Asia Pacific
輸入月流量 GB 數即可
🔀
Application Load Balancer
Compute · LCU 計費
月費 = 固定費 + LCU 費
固定費 = $0.008/hr × 720hr = $5.76
LCU = max(連線數, 請求數, 頻寬, 規則)
# 一般情境 LCU ≈ $15–20
💡
LCU 是什麼？
Load Capacity Unit，取四個維度最高值計費：
新連線數 / 每秒活躍連線 / 頻寬 / 評估規則數
🎯
計算機設定
Load balancer type: Application
輸入預估 HTTP 請求數（萬/月）
🛡️
NAT Gateway
Compute · 固定費+流量費
月費 = 固定費 + 資料處理費
固定費 = $0.045/hr × 720hr = $32.4
資料處理 = 流量(GB) × $0.045/GB
區域 NAT 閘道（必填）= 額外固定費
# 80GB → $32.4 + $3.6 + 區域NAT = $50.22
⚠️
重要概念
NAT GW 有固定費 $32/月，即使流量很少也要付。
EC2 連 S3 建議改用 VPC Gateway Endpoint（免費）省去 NAT 費用。
📐
架構說明
NAT GW 僅處理「出站」流量（Private EC2 → Internet）。
入站流量走 IGW → ALB，完全不經過 NAT。
🖥️
Amazon EC2 t3.medium
Compute · 按時計費
架構共 4 台 EC2（WEB Tier×2 + AP Tier×2）
月費 = EC2費 + EBS費（合併計費）
WEB×2：EC2 $0.0544×720×2 + EBS 50GB×$0.096×2 = $89.02
AP×2 ：EC2 $0.0544×720×2 + EBS 50GB×$0.096×2 = $89.02
# EC2+EBS 合計 $178.04（含儲存，與 Pricing Calculator 一致）
💡
架構說明
WEB Tier EC2（Private Subnet）：處理 HTTP 請求，接收 ALB 分流。
AP Tier EC2（Private Subnet）：處理商業邏輯，呼叫 Aurora PostgreSQL DB。
兩層各部署在 AZ-A、AZ-B，實現 Multi-AZ 高可用。
🎯
計算機設定
Instance type: t3.medium
Pricing: On-Demand · 設定每天使用時數
💾
Amazon EBS gp3
Storage · 全月計費
月費 = 容量(GB) × $0.08/GB × 台數
架構共 4 台 EC2，掛載 EBS 各 50GB
= 已含於各 EC2 費用中（50GB × $0.096/台，2台/Tier）
# EC2 停機不省 EBS 費！
⚠️
注意事項
EBS 依「掛載容量」全月計費，EC2 關機不影響 EBS 費用。
若要停用請先 Snapshot 再刪除 Volume。
💡
gp3 特色
3,000 IOPS 基礎免費，無需額外付費。
比 gp2 便宜約 20%，銀行一般負載適用。
🛢️
Amazon Aurora PostgreSQL
Database · 架構最高費用 · PostgreSQL
月費 = 實例費 + 儲存費 + I/O費
實例 = $0.320/hr × 720hr × 2台 = $460.8
儲存 = 120GB × $0.10/GB = $12
# AWS 實際費率 $0.320/hr（CSV反推），Primary + Replica 各自計費
合計 $472.01（CSV 完全一致）
⚠️
為何最貴？
db.r6g.large 是記憶體優化型（16GB RAM），全月運行費高。
通常是整架構費用最高的單一服務。
📐
架構說明
放在 Isolated Subnet（紅色框），無 Internet 路由，
只允許同 VPC 內 EC2 透過 TCP 5432 連線。
🪣
Amazon S3 Standard
Storage · Region 級服務
月費 = 儲存費 + 請求費
儲存 = 300GB × $0.025/GB = $7.5
GET/PUT 請求費 ≈ $0（免費層）
合計約 $7.50
📐
架構說明
S3 是 Region 級服務，放在 VPC 外。
使用 VPC Gateway Endpoint 讓 EC2 私密存取 S3，
不需經過 NAT Gateway（節省 NAT 費用）。
💡
省費技巧
低頻存取的報表/PDF 可改用 S3 Standard-IA，
儲存費用約省 40%（$0.0138/GB）。
